## Top 3 Things to Fix

1. **Double navbar on homepage** — `page.tsx` imports `<Navbar />` which duplicates the one in `layout.tsx` [high]
2. **growth-yield.mp4 is 15.8MB** — CTA section video is too heavy for web delivery [high]
3. **3 missing listing-specific video previews** — 404 errors on `/launchpad` for per-listing preview videos [medium]

## All Issues

### Critical
None.

### High

**ISSUE-001: Double navbar on homepage**
- Source: browser
- Page: `/` (homepage only)
- Evidence: Two identical navbars stacked at top of page. Visible on both desktop and mobile (two hamburger menus).
- Root cause: `app/layout.tsx:35` renders `<Navbar />` globally for all pages. `app/page.tsx:125` also renders `<Navbar />` inside the page component.
- Fix: Remove `import Navbar` and `<Navbar />` from `app/page.tsx`. The layout already handles it.
- Difficulty: trivial (2-line delete)

**ISSUE-002: growth-yield.mp4 is 15.8MB**
- Source: static-analysis
- File: `public/video/growth-yield.mp4`
- Evidence: 15.8MB for a background video is too heavy. Other videos are 2-7MB which is reasonable.
- Fix: Re-encode at lower bitrate or shorter loop. Target < 5MB.
- Difficulty: easy

### Medium

**ISSUE-003: Missing listing-specific video previews (3x 404)**
- Source: browser (console errors on `/launchpad`)
- Evidence: `LaunchpadCard.tsx:74` tries `/video/preview-{listing.id}.mp4` for each listing. These files don't exist: `preview-e445f791-...`, `preview-7bed8728-...`, `preview-9de079a0-...`
- The fallback `preview-generic.mp4` exists but browser still logs 404 for the primary source.
- Fix: Suppress 404 by checking file existence before setting src, or use the generic video as sole source.
- Difficulty: easy

**ISSUE-004: Sour Diesel hero image shows vegetables**
- Source: browser
- Page: `/launchpad`
- Persisting from previous audit. Supabase data fix needed, not code.
- Difficulty: trivial (data fix)
